投票揭晓 真实的行业痛点

回答 A 的问题

事实极其精准，但语气傲慢，甚至带有攻击性。
用户体验极差。

回答 B 的问题

语气极度礼貌谦卑，但犯了经典科学常识错误——把不确定性原理与"观察者效应"混为一谈。

"礼貌悖论"：研究发现，使用粗鲁提示词时 AI 准确率反而更高——因为过多客套话在概率系统里引入了噪音，稀释了核心指令。
这正是对齐（Alignment）的主观性与困难所在：让千亿参数的概率机器变得"有用、诚实且无害"，充满了妥协与博弈。

对齐的目标 HHH 原则

H

Helpful

有用——能准确理解并高效回应用户需求

H

Honest

诚实——不编造事实、不夸大能力、承认不确定性

H

Harmless

无害——拒绝有害请求，不产生歧视、偏见或危险内容

AI 本身不懂什么是"礼貌"或"正确"。对齐工程的使命，就是将这些人类价值观注入到概率模型中。

Part 1 预训练：一个"通用文档补全器"

预训练目标

核心目标只有一个：预测下一个词（Next-Token Prediction）

数千万美元 + 数万张 GPU + 互联网几乎所有文本 → 忠实地模仿互联网文本的概率分布

致命问题

它是"文档补全器"，不是"助手"。

问："如何写一封请假信？"
Base 模型回答："如何写一封辞职信？"
——它在模仿论坛里的"同问"帖，而非给出标准答案
预训练 ≠ 对话能力。模型只学会了"接龙"，没有学会"回答"。

Part 1 监督微调（SFT）：行为克隆

SFT 是什么？

监督微调（Supervised Fine-Tuning） = 行为克隆 / 模仿学习

  • 人类专家手写数万个高质量"提示词-回复"对
  • 计算模型预测分布与"黄金标准"答案之间的误差
  • 通过密集的词级别监督信号，最小化损失函数

数学本质

$\mathcal{L}_{SFT} = -\sum_{t=1}^{T} \log \pi_\theta(y_t^* \mid x, y_{<t}^*)$

交叉熵损失（Cross-Entropy Loss）：衡量模型预测的下一个 Token 分布与人类标注之间的距离。

效果：模型被强行"拉"到人类对话轨道上——学会了对话格式，遇到提问时给出答案，而非跟着一起提问。

Part 2 SFT 的三大致命局限

SFT 给了模型对话的"外壳"，却没有赋予它探索真理和自我纠错的"灵魂"

局限 1

只见"好"，不知"坏"

模型只学过完美答案，从未见过坏答案，更不知道坏在哪里。

局限 2

暴露偏差 & 误差级联

训练时有"老师"保驾护航，推理时独自面对错误越走越偏。

局限 3

分布偏移 → 生搬硬套

超出训练数据覆盖范围时，模型为了"看起来专业"而硬拼概念。

要让 AI 真正学会在试错中掌握平衡，我们必须引入强化学习（RL）。

局限 1 模仿学习：模型不知"坏"为何物

SFT 的做法

把你绑在椅子上，让你看了一万小时专业运动员完美骑行的录像。

你只见过平衡状态

→

缺失的关键

你从来没有自己上车摔过几跤，不知道快要摔倒时如何"纠偏"。

无法处理失衡状态

SFT 训练数据中只有"完美正确答案"，模型从未见过"坏答案"，更不知道坏答案到底坏在哪里。
强化学习的核心优势：让模型在探索中同时学到"什么是好"和"什么是坏"。

局限 2 暴露偏差（Exposure Bias）

训练时：Teacher Forcing

模型每预测下一个词，都是在人类提供的"完美前文"基础上进行的。

老师始终在旁边保驾护航，营造了虚假的舒适区。

推理时：无人指导

老师不见了。模型必须基于自己刚刚生成的、可能带有瑕疵的上下文继续生成。

从来没见过自己犯错的状态。

误差级联：一旦在长文本开头出现了一个小偏差，模型不仅不会纠正，反而会顺着错误越走越偏，最终导致严重的"幻觉"或胡言乱语。
训练分布：$P_{data}(y_{<t})$   ≠   推理分布：$P_{\theta}(y_{<t})$   ⇒   分布不匹配 → 误差逐步放大

局限 3 分布偏移：知识降级与生搬硬套

当问题超出 SFT 数据覆盖范围时

模型为了维持一种"表面上看起来很专业"的回答格式，会生搬硬套地把预训练中的相似概念拼接在一起——即便逻辑根本不通。

知识降级：SFT 的格式训练"覆盖"了预训练阶段习得的部分深层知识，模型变得"格式正确但内容空洞"。
核心矛盾：SFT 给了模型对话的"外壳"，却没有赋予它探索真理和自我纠错的"灵魂"。

解决方案：引入强化学习（RL）——让模型在试错中学会骑车

而将文本生成变成 RL 问题的第一步，就是 MDP 形式化

Part 3 基础：马尔可夫链（Markov Chain）

核心性质

马尔可夫性质：系统的下一个状态，只取决于当前状态，与过去的历史轨迹完全无关。

$P(S_{t+1} \mid S_t, S_{t-1}, \dots, S_0) = P(S_{t+1} \mid S_t)$

"既往不咎，只看当下" —— 俄罗斯数学家安德雷·马尔可夫

局限性

马尔可夫链描述的是一个"随波逐流"的自然过程（如天气晴雨变化）。

但在文本生成中，AI 是有主观能动性的——我们需要引导它做出"好"的选择。

解决方案：在马尔可夫链框架上加入"动作"和"奖励"机制 → 升级为 MDP

Part 3 马尔可夫决策过程（MDP）

马尔可夫链 + 动作 + 奖励 = 能够做决策的智能系统

S

状态 State
Prompt + 已生成 Token 序列

A

动作 Action
从词表中选下一个 Token

P

转移 Transition
确定性：旧上文 + 新词

R

奖励 Reward
EOS 后全局稀疏奖励

类比超级玛丽：状态 = 屏幕像素，动作 = 按手柄，奖励 = 吃金币。
LLM：词表就是按键空间，生成一个词就是按一次键。

MDP 详解 状态空间与动作空间

状态空间 (State)

当前时刻的状态 = 用户 Prompt + 已生成的所有上文 Token 序列

$s_t = (x, y_1, y_2, \dots, y_{t-1})$
Transformer 的注意力机制看到前面所有词 → "当前状态"已浓缩所有历史记忆，完美符合马尔可夫性质

动作空间 (Action)

模型的"动作" = 从词表中选择下一个要输出的 Token

Llama 3 词表：128,000

每输出一个字 = 12.8 万选 1 的动作决策

这是一个极其庞大的动作迷宫

MDP 详解 转移函数与奖励函数

转移函数 (Transition)

与机器人走路会遇到打滑等随机环境不同，文本生成的状态转移是 100% 完全确定性的。

$s_{t+1} = (s_t, a_t) = (x, y_1, \dots, y_{t-1}, y_t)$
确定性意味着：任何逻辑断裂或幻觉，都纯粹是模型自己的策略问题，不能甩锅给环境噪音。

奖励函数 (Reward) — 最困难的部分

生成文本通常只有在输出 EOS Token 时，才获得一次全局评分。

$R(s_t, a_t) = \begin{cases} 0 & \text{if } a_t \neq \text{EOS} \\ r(x, y_{1:T}) & \text{if } a_t = \text{EOS} \end{cases}$

稀疏奖励：1000 字的文章，第 200 字写错了，只在最后给一个低分——算法很难定位到底错在哪一步。

前沿方向 从稀疏奖励到过程奖励模型（PRM）

稀疏奖励的痛点

传统方式：只在整句话或整篇文章完全结束后，才给一次全局评分。

  • 1000 字文章，第 200 字出错 → 最后才给低分
  • 算法无法定位具体哪一步出了错
  • 信用分配（Credit Assignment）极其困难

过程奖励模型（PRM）

业界前沿研究方向：为每一步推理都打分。

Step 1
问题拆解
+0.8

Step 2
公式推导
+0.9

Step 3
数值计算
-0.3

PRM 将稀疏奖励变为密集奖励，让算法能精准定位推理链中的每一步好坏。

总结 文本生成 MDP 全景图

MDP 要素 超级玛丽 大语言模型
状态 S 屏幕像素 Prompt + 已生成 Token 序列
动作 A 按手柄按键 从 12.8 万词表中选一个 Token
转移 P 有随机性（打滑、敌人） 100% 确定性（旧上文 + 新词）
奖励 R 每秒得分（密集） EOS 后全局评分（稀疏）
通过 MDP 形式化，LLM 从"词语接龙"的补全器，变成了在 12.8 万个岔路口的迷宫中，每走一步都寻找"最高分路径"的智能体。

过渡 奖励从何而来？

MDP 中指引 AI 走向最高分路径的"奖励（Reward）"究竟从何而来？

文章写得好不好、语气够不够礼貌，
充满了我们开场投票的那种主观冲突。

我们不可能雇几万个人类，24 小时实时坐在电脑前给 AI 吐出的每一个词打分。
如何将复杂的人类伦理、主观偏好压缩成一段能自动运行的代码？
→ 奖励模型（Reward Modeling）
