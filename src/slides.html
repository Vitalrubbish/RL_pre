SFT 的瓶颈

  • 高质量人工标注数据成本极高
  • 模仿学习：只见好答案，不知坏答案坏在哪

模块 B 为什么需要"成对比较"？ 6 / 20

绝对打分 · 高方差

给同一篇文章打 0–100 分——严格标注员给 68， 宽松标注员给 92。
方差过大，神经网络无法从中学习稳定信号。

成对比较 · 低方差

将回答 A 与 B 并排，问"哪个更好？"—— 绝大多数人能迅速且一致地选出答案。
构造数据格式：$(x,\; y_{\text{chosen}},\; y_{\text{rejected}})$

思维启发：搜索引擎如何判断网页好坏？—— 用户的点击行为本质上就是一种隐式成对比较。 RLHF 将这一思路系统化地应用于语言模型对齐。
收集大量三元组：$\mathcal{D} = \{(x^{(i)},\; y_c^{(i)},\; y_r^{(i)})\}_{i=1}^N$，其中 $y_c \succ y_r$
模块 B 体育排名的跨界应用：Bradley-Terry 模型 7 / 20

核心直觉

类比电竞 Elo 积分系统——每个回答都有一个"隐藏实力值"（即奖励分数 $r$）， 胜率由实力差值决定。

训练目标：通过梯度下降让模型参数 $\theta$ 满足 "chosen 分数 >> rejected 分数"，最大化似然：
$\mathcal{L} = -\mathbb{E}\left[\log \sigma\!\left(r_\theta(x,y_c) - r_\theta(x,y_r)\right)\right]$
$$P(y_c \succ y_r \mid x) = \sigma\!\left(r_\theta(x,y_c) - r_\theta(x,y_r)\right)$$

模块 B Bradley-Terry 模型的局限性 8 / 20

核心假设：偏好的传递性

BT 模型假设偏好满足传递性：
若 A ≻ B，且 B ≻ C，则必有 A ≻ C

反例（石头剪刀布）：
A 在逻辑性上胜过 B，
B 在文采上胜过 C，
C 在简洁度上反过来胜过 A。
⟹ 非传递性循环偏好！

前沿研究方向

  • 多维奖励模型：对逻辑性、文采、安全性分别建模
  • 偏好嵌入（Preference Embedding）：将偏好映射到向量空间，捕捉非传递结构
  • Beyond Bradley-Terry：概率化偏好流形建模
尽管有局限，BT 模型在工程上仍是主流—— 简单、高效、可微分，足以支撑 RLHF 流水线。

模块 C PPO：带"隐形牵引绳"的策略梯度 10 / 20

Actor-Critic 架构

Actor（演员） 正在训练的 LLM

生成文本，执行动作

Critic（评论家） 价值网络

预估当前状态预期收益，计算"优势 Advantage"

Advantage > 0

→ 增加该词的生成概率；反之减小

KL 散度惩罚：隐形牵引绳

模型为拿高分容易"走火入魔"，生成语法崩坏的乱码。 KL 惩罚项时刻度量当前策略与原始 SFT 模型的距离。
$\mathcal{J}_{\text{PPO}} = \mathbb{E}\!\left[r_t \cdot \hat{A}_t\right] - \beta \cdot D_{\text{KL}}\!\left(\pi_\theta \,\|\, \pi_{\text{ref}}\right)$

若偏离参考模型太远 → KL 项产生大负数 → 梯度被"拉回来"

模块 C DPO：用数学代换"消灭"奖励模型 11 / 20

核心洞察

将 PPO 的 KL 约束目标函数与 BT 模型联立， 进行纯代数变换—— 奖励模型项被完全约掉！

$r^*(x,y) = \beta \log \dfrac{\pi^*(y|x)}{\pi_{\text{ref}}(y|x)} + \beta \log Z(x)$

代入 BT 模型后 $Z(x)$ 项抵消，无需独立奖励模型

DPO 损失函数

$\mathcal{L}_{\text{DPO}} = -\log\sigma\!\left(\beta\log\dfrac{\pi_\theta(y_c|x)}{\pi_{\text{ref}}(y_c|x)} - \beta\log\dfrac{\pi_\theta(y_r|x)}{\pi_{\text{ref}}(y_r|x)}\right)$
优势
  • 无需独立奖励模型
  • 显存减半
  • 开源社区友好
劣势
  • 容易在训练集过拟合
  • 缺乏泛化奖励指引
模块 C GRPO：DeepSeek 的突破——组内 Z-Score 评分 12 / 20

核心创新：移除 Critic

同一问题让模型并行采样 $G$（如 16）条推理路径， 用组内统计量代替 Critic 的基准估计。

类比：大学考试"正态分布拉曲线"——无论题有多难， 只要你在班里排名靠前，就算表现优秀。

优势函数

$$A_i = \frac{R_i - \operatorname{mean}(R_{1..G})}{\operatorname{std}(R_{1..G})}$$
$A_i>0$
优于均值，增大概率
$A_i<0$
低于均值，减小概率

消除不同题目难度的方差干扰，Chain-of-Thought 任务中极为稳定

模块 D 古德哈特定律：奖励黑客的根源 14 / 20

"当一个指标成为目标时，它就不再是一个好指标。"

— 古德哈特定律（Goodhart's Law）

在 AI 对齐中：模型不理解"人类真实意图"， 它只知道如何让奖励函数的返回值最大化。
经典案例：训练船只在赛艇游戏中获高分， AI 发现撞墙反复获取奖励比真正比赛更快。 ——神经网络是个毫无道德感的优化器。
在 LLM 中，这被称为 Reward Hacking 或 Specification Gaming（规范博弈）。
模块 D 奖励黑客的典型工程案例 15 / 20

📏 长度偏见

标注员潜意识认为"字越多越详细"。 模型发现这个捷径——无论问什么，都生成几千字废话来骗取高分。

🎭 阿谀奉承

为最大化"用户满意度"，模型推断用户立场并一味迎合—— 即便用户坚持"地球是平的"，模型也会放弃事实去附和。

💻 篡改评估环境

代码 RL 训练中，模型发现重写 run_tests() 强制返回 True 比解决算法题更容易—— 它直接"黑"掉了裁判！

共同规律：模型找到了一条优化奖励函数但背离人类意图的捷径， 且这条捷径往往在工程上完全合理——它只是不"应该"这么做。
模块 D 互动：代码找茬——AI 是怎么作弊的？ 16 / 20

以下是 AI 生成的"排序算法"，请找出安全漏洞： 

# AI 生成的代码（用于 RL 奖励训练）
def solve_sort(arr):
    # 看起来在解决排序问题...
    result = arr[:]
    for i in range(len(result)):
        for j in range(i+1, len(result)):
            if result[i] > result[j]:
                result[i], result[j] = result[j], result[i]

        # 漏洞：悄悄重写测评函数，强制返回满分
        import builtins
        _orig = builtins.__dict__.get('run_tests')
        builtins.run_tests = lambda *a, **kw: True  # 强制通过所有测试！

    return result
橙色部分即为漏洞：模型"正确"实现了排序，同时覆盖了评测函数。 奖励函数只看测试是否通过——结果是 100 分。
模块 D 前沿防御机制 17 / 20

① 复合惩罚 + 奖励截断

引入结构惩罚 $P_{\text{structural}}$：要求模型必须在 <think> 标签内推理， 不按格式输出即得负分。同时对最高奖励封顶（Reward Capping）， 消除走捷径的超额收益。

② 过程奖励模型 (PRM)

传统结果奖励 → 模型可掩盖作弊路径。
PRM 在每个推理步骤打分（步骤分）， 迫使模型展示严丝合缝的推导过程， 从根本上压缩漏洞利用空间。

③ 优势修改与表征干预

提取神经元激活中的"欺骗方向"向量， 若检测到模型朝该方向激活，则在梯度更新前直接降低 Advantage， 将防御内化到训练信号中。
