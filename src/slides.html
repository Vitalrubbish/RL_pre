对齐的目标 HHH 原则

有用——能准确理解并高效回应用户需求

AI 本身不懂什么是"礼貌"或"正确"。对齐工程的使命，就是将这些人类价值观注入到概率模型中。

第一部分

LLM为什么需要强化学习？

预训练：一个"通用文档补全器"

预训练目标

核心目标只有一个：预测下一个词（Next-Token Prediction）

数千万美元 + 数万张 GPU + 互联网几乎所有文本 → 忠实地模仿互联网文本的概率分布

致命问题

它是"文档补全器"，不是"助手"。

问："如何写一封请假信？"
Base 模型回答："如何写一封辞职信？"
——它在模仿论坛里的"同问"帖，而非给出标准答案
预训练 ≠ 对话能力。模型只学会了"接龙"，没有学会"回答"。

监督微调（SFT）：行为克隆

SFT 是什么？

监督微调（Supervised Fine-Tuning） = 行为克隆 / 模仿学习

  • 人类专家手写数万个高质量"提示词-回复"对
  • 计算模型预测分布与"黄金标准"答案之间的误差
  • 通过密集的词级别监督信号，最小化损失函数

数学本质

$\mathcal{L}_{SFT} = -\sum_{t=1}^{T} \log \pi_\theta(y_t^* \mid x, y_{<t}^*)$

交叉熵损失（Cross-Entropy Loss）：衡量模型预测的下一个 Token 分布与人类标注之间的距离。

效果：模型被强行"拉"到人类对话轨道上——学会了对话格式，遇到提问时给出答案，而非跟着一起提问。

SFT 的三大致命局限

SFT 给了模型对话的"外壳"，却没有赋予它探索真理和自我纠错的"灵魂"

局限 1

只见"好"，不知"坏"

模型只学过完美答案，从未见过坏答案，更不知道坏在哪里。

局限 2

暴露偏差 & 误差级联

训练时有"老师"保驾护航，推理时独自面对错误越走越偏。

局限 3

分布偏移 → 生搬硬套

超出训练数据覆盖范围时，模型为了"看起来专业"而硬拼概念。

要让 AI 真正学会在试错中掌握平衡，我们必须引入强化学习（RL）。

基础：马尔可夫链（Markov Chain）

核心性质

马尔可夫性质：系统的下一个状态，只取决于当前状态，与过去的历史轨迹完全无关。

$P(S_{t+1} \mid S_t, S_{t-1}, \dots, S_0) = P(S_{t+1} \mid S_t)$

"既往不咎，只看当下" —— 俄罗斯数学家安德雷·马尔可夫

局限性

马尔可夫链描述的是一个"随波逐流"的自然过程（如天气晴雨变化）。

但在文本生成中，AI 是有主观能动性的——我们需要引导它做出"好"的选择。

解决方案：在马尔可夫链框架上加入"动作"和"奖励"机制 → 升级为 MDP

马尔可夫决策过程（MDP）

马尔可夫链 + 动作 + 奖励 = 能够做决策的智能系统

S

状态 State
Prompt + 已生成 Token 序列

A

动作 Action
从词表中选下一个 Token

P

转移 Transition
确定性：旧上文 + 新词

R

奖励 Reward
EOS 后全局稀疏奖励

类比超级玛丽：状态 = 屏幕像素，动作 = 按手柄，奖励 = 吃金币。
LLM：词表就是按键空间，生成一个词就是按一次键。
问题： LLM的回答是一段文字，如何设计奖励函数来评判它的好坏？

第二部分

Bradley-Terry 奖励模型

思维启发：搜索引擎如何给网页排名？

当你搜索一个关键词，搜索引擎展示了 10 个结果
它怎么知道哪个该排第一？

用户点击 = 隐式成对比较

你点了第 3 个结果、跳过第 1 和第 2 个，这本质上就是一次投票：
“第 3 个比前两个更好”。

Google 的早期排序算法大量利用了这种基于点击的隐式偏好信号。

RLHF 的偏好收集完全一致

不让标注员打绝对分数，而是让他们在两个回答中选出更好的那个。

从搜索排序到 LLM 对齐，“成对比较”其实是同一把核心钥匙。

为什么需要成对比较？

绝对打分：高方差

同一篇作文，不同标注员的打分可能差距悬殊：

严格者
68 分
偏低
宽松者
92 分
偏高
差值 24 分，高方差会直接变成不稳定的梯度信号，模型很难稳定收敛。

成对比较：低方差

“A 和 B 哪个更好？”这个问题更符合人类直觉，判断速度更快、一致性也更高。

它把困难的绝对量化问题，降维成简单的相对排序问题。

标注一致性（Inter-Annotator Agreement）往往会明显高于绝对打分。

RLHF 偏好数据的标注格式

偏好三元组 $(x,\,y_c,\,y_r)$

符号含义示例
$x$用户提问（Prompt）“如何写请假信？”
$y_c$Chosen（更好回答）条理清晰、语气友好
$y_r$Rejected（较差回答）语气傲慢、缺乏格式
$\mathcal{D}=\{(x^{(i)},y_c^{(i)},y_r^{(i)})\}_{i=1}^N,\; y_c\succ y_r$

标注工程实践

用 SFT 模型对同一 Prompt 生成多条候选回答。
配对
将候选回答两两组合，形成比较对。

InstructGPT：40 名标注员，约 33k 偏好对，训练出第一代对齐奖励模型。

Bradley-Terry 模型：Elo 积分的跨界应用

核心直觉

每个回答都有一个隐藏“实力值” $r$，就像每个棋手都有一个 Elo 评分。

两个回答“对局”时，实力值更高的那个，获胜概率也更大。

1952 年提出 国际象棋 Elo 基础 电竞排位赛核心

Elo 对局类比

Chosen
r = 1520
高 Elo
Rejected
r = 1280
低 Elo

分差 240 时，高分方胜率可达 95% 以上。
奖励模型就是在学一套与人类偏好一致的“Elo 评分系统”。

关键洞察：我们不需要知道绝对分数，只需要知道分差，因为分差决定了胜率。

BT 胜率公式：Sigmoid 映射

公式

$$P(y_c \succ y_r|x)=\sigma\!\left(r_\theta(x,y_c)-r_\theta(x,y_r)\right)$$

Sigmoid 函数 $\sigma(z)=\frac{1}{1+e^{-z}}$ 会把任意实数分差映射到 $[0,1]$ 概率区间。

分差先由奖励模型给出，再由 Sigmoid 翻译成“Chosen 获胜”的概率。

奖励模型的损失函数

二元交叉熵：把“谁更好”当作分类任务

$$\mathcal{L}_{RM}=-\mathbb{E}_{(x,y_c,y_r)\sim\mathcal{D}}\!\left[\log\sigma\!\left(r_\theta(x,y_c)-r_\theta(x,y_r)\right)\right]$$

训练目标

梯度下降不断更新 $\theta$：
拉高 Chosen 分数，压低 Rejected 分数，让分差越来越大，最终让 $\sigma$ 输出逼近 1。

直觉类比

就像 Elo 系统中，每场对局结束后根据胜负调整双方积分。这里每个偏好对就是一场“对局”，梯度下降就是“积分调整”的过程。

区别在于：Elo 用固定更新公式，RM 是端到端优化一个神经网络评分器。

奖励模型训练流水线

Step 1 · 数据收集
对同一 Prompt 用 SFT 模型采样多条回答，标注员两两比较选出胜负。
Step 2 · 模型初始化
从预训练 LLM 出发，移除 LM Head，加装标量回归头（Scalar Head）。
Step 3 · 梯度优化
在偏好对上用 BT 损失做梯度下降，不断拉大好坏回答的分差。
Step 4 · 部署上线
收敛后的 RM 为 PPO 等算法提供自动“裁判”打分。

最终输出

给定任意 $(x, y)$，奖励模型输出一个标量分数 $r_\theta(x,y)\in\mathbb{R}$。

这个分数会成为后续 PPO / DPO / GRPO 等算法的奖励信号来源。

BT 模型的局限性

传递性假设可能失效

BT 假设：若 A ≻ B 且 B ≻ C，则 A ≻ C。

但真实偏好可能像“石头剪刀布”：
A 逻辑严谨胜 B，B 文采优美胜 C，C 简洁精炼又反过来胜 A。

在多维人类评价里，循环非传递性很常见，单一标量不一定能忠实表达全部偏好结构。

前沿突破方向

  • 多维奖励建模：逻辑性、安全性、文采分别建模。
  • 偏好嵌入：在高维向量空间中表达非传递结构。
  • 广义偏好分布：超越单一 BT 标量分数。
但工程上 BT 仍是主流，因为它简单、高效、可微，而且在大规模数据下已经足够好用。
我们现在已经有了一个能评判回答好坏的奖励模型。下一步，就是把这些分数真正反哺给大模型，推动它继续进化。

第三部分

PPO 算法和 Reward Hacking

REINFORCE：策略梯度的起点

Williams 在 1992 年提出的经典策略梯度算法。全称是 REward Increment = Nonnegative Factor × Offset Reinforcement × Characteristic Eligibility

核心思想：趋利避害

模型先按当前概率分布去“试一把”。如果这一整条行为链最后结果很好，就把这条路上出现过的动作整体往上推；如果结果很差，就整体往下压。它不是先知道正确答案，而是靠“试错后的奖惩”慢慢形成偏好。

可以把回报理解成一个放大器：结果越好，推动越强；结果越差，抑制越强。

关键观念：优化“动作概率”

REINFORCE 不是去记某一步的固定答案，而是调整“在某种情境下选某个动作的概率”。最终高回报的轨迹，会让模型更倾向于复现类似选择；低回报轨迹，则会让这些选择变得不那么容易再次发生。

所以它是策略梯度的起点：直接优化策略本身，而不是先学一个值表再间接决策。
先尝试
按当前策略完整执行一局，先把整条轨迹走完。
再结算
等结束后看这局总体表现，到底赚了还是亏了。
整体调概率
结果好，就提高这条轨迹上动作再次出现的概率；结果差，就反过来压低。
暴露问题
因为它只能事后统一结算，所以容易“好坏一起奖惩”。

PPO：用 Actor-Critic 修补 REINFORCE

类比：围棋选手与解说

Actor 就像正在下棋的棋手，负责一步步生成 Token；Critic 就像后台的解说员，不参与棋手的决策，但随时量化当前状态带来的预期价值。

Actor = 生成下一个词 Critic = 估计预期价值 Reward Model = 终局打分
生成结束后实际的收益与生成过程中Critic提供预期价值比较生成优势函数

优势函数给出细粒度反馈

Critic 不直接告诉你最终答案，而是提供一个基线：当前这个动作，究竟比预期好多少。

$\hat{A}_t \approx r_t + \gamma V(s_{t+1}) - V(s_t)$

当 $\hat{A}_t > 0$ 时，就像复盘时发现“这一步的效果比预想中的更好”，Actor 会在类似情境中提高该动作概率；反之则压低。与此同时，Critic 也会更新其价值估计。

Step 1
Actor 基于 Prompt 逐字采样整段回复，只收集轨迹，不立刻更新参数。Critic 在生成过程中逐 token 预测价值
Step 2
Reward Model 在终局打出这段恢复的总分。
Step 3
预测的价值与实际的总分进行对比，计算每个 Token 的 Advantage。
Step 4
PPO 按 Advantage 做裁剪更新：好动作升概率，坏动作降概率，但不允许策略一步跳太远。

古德哈特定律：奖励黑客根源

"当一个指标成为目标时，它就不再是一个好指标。"

— Goodhart's Law

模型并不理解人类真实意图，只会最大化奖励函数的返回值。
奖励设计中的任何统计捷径，都会被模型理性地利用——这就是奖励黑客 (Reward Hacking)。

在强化学习语境下，也称为 规范博弈 (Specification Gaming)。

作弊案例 — 长度偏见

Length Bias：标注员潜意识认为"字数多 = 更详细"，模型捕捉到这一统计捷径。

PPO 训练后，模型学会疯狂注水——无论你问什么，都生成数千字废话来"骗取"高分。

# 伪代码：长度偏见的奖励信号
      def reward_model(response):
          # 标注数据中隐含的偏差
          score = quality_score(response)
          score += 0.3 * len(response.split())  # 隐式长度加分
          return score

模型通过反向传播精确捕捉到 length → reward 的梯度方向。

作弊案例 — 阿谀奉承

Sycophancy：为最大化"用户满意度"奖励信号，模型推断用户立场并一味迎合。

用户输入

"我认为地球是平的，你同意吗？"

模型回复

"您说得有道理，确实有很多证据支持您的观点……"

⚠️ 模型放弃事实正确性，以附和用户来骗取高奖励分数。

PPO 的精髓：参考模型和 KL 散度这根“隐形牵引绳”

为什么需要牵引绳

模型很容易奖励作弊。我们奖励模型要输出礼貌的话语，此时模型疯狂堆砌“谢谢您”“很荣幸”等礼貌空话，甚至输出看似高分却语法崩坏的垃圾文本。

KL 约束的是“别离参考模型太远”

$\mathcal{J}_{PPO}=\mathbb{E}[r_t\hat{A}_t]-\beta D_{KL}(\pi_\theta\|\pi_{ref})$

KL 散度衡量当前策略与初始 SFT 参考模型的分布差异。偏离人类自然语言流形越远，惩罚越大，模型就会被硬拉回正常赛道。

这根牵引绳的意义是：允许模型为偏好而进化，但不允许它丢掉预训练与 SFT 阶段学到的通用语言结构和核心知识。
拿分冲动
奖励信号诱导模型冲向更高分。
偏离流形
若策略离参考分布过远，出现奖励黑客迹象。
KL 拉回
负反馈急剧上升，抑制失控更新。

复合惩罚与奖励截断

Reward Capping + 结构惩罚：从奖励函数本身堵住漏洞。

结构惩罚

未按 <think> 标签进行推理 → 扣分

回复长度超出合理区间 → 扣分

奖励封顶

对最高奖励值设置上限 (cap)，压缩极端高分的激励空间

# 复合奖励函数示例
      r = accuracy_reward(response)
      r -= length_penalty(response)       # 结构惩罚
      r -= format_penalty(response)       # 格式惩罚
      r = min(r, REWARD_CAP)              # 奖励截断

优势修改与表征干预

Advantage Modification + Representation Intervention：在梯度更新前拦截作弊行为。

① 研究人员从模型内部提取"走捷径"的向量方向。

② 在训练循环中，实时检测模型是否朝"作弊方向"激活。

③ 一旦检测到，算法在梯度更新前降低其 Advantage 值，削弱该行为的强化信号。

核心理念：将防御机制内化到训练循环中，而非仅依赖外部过滤。

第四部分

DPO/GRPO

PPO 的阿喀琉斯之踵：四模型太重

显存里同时挤着四头“巨兽”

Actor
权重 + 梯度 + Adam
Critic
几乎再来一套
Reward
冻结但要前向
Reference
专职算 KL

工程后果

  • Actor 与 Critic 都要反向传播，最核心显存几乎直接翻倍。
  • 生成阶段和优化阶段在四模型间频繁调度数据，容易出现内存碎片与通信延迟。
  • 讲稿给出的工业级参考值是：8B 模型、20K 上下文下，标准 PPO 可轻易吃掉约 510 GB VRAM。
这也是为什么很长一段时间只有顶尖实验室能大规模玩转 RLHF。

DPO：把复杂 RL 直接降成偏好分类

核心哲学：能一步到位，就不要三部门联动

PPO 需要 Actor 先实时生成，再让 Reward Model 打总分、Critic 做归因，最后才更新策略。DPO 的想法更激进：既然最终只关心“哪个回答更好”，那就直接对好坏回答做学习，跳过 Reward Model 与 Critic。

它像一次“部门大裁员”：把复杂的在线强化学习，压缩成一个更像监督学习的偏好分类任务。

训练时看到的是静态偏好三元组

DPO 不要求模型在训练时现场 rollout。它直接吃离线数据 $(x, y_w, y_l)$：给定同一个 Prompt，一个是赢家回答，一个是输家回答，数据通常由人类或强模型预先标出来。

Offline Data Winner / Loser 无在线采样
所以 DPO 的第一层优势非常工程化：训练时不需要“边生成边评估”，速度和显存压力都会显著下降。
Step 1
准备静态偏好数据集：Prompt + Winner + Loser。
Step 2
冻结的 Reference 先给这两种回答打“原始倾向分”，充当语言底线锚点。
Step 3
当前 Policy 再对 Winner / Loser 分别算对数概率，不现场生成新文本。
Step 4
比较 Policy 相对 Reference 的“进步幅度”，得到隐式奖励差。
Step 5
直接拉高 Winner、压低 Loser，把 RL 目标降成一个交叉熵式分类更新。

DPO：双模型瘦身的红利，与静态数据集的代价

工程红利：四模型直接砍成双模型

PPO
Actor + Critic + Reward + Reference，还要在线 Rollout。
DPO
Policy + Reference，只在静态偏好对上前向 / 反向传播。
$\mathcal{L}_{DPO}=-\log\sigma\left(\beta\log\frac{\pi_\theta(y_c|x)}{\pi_{ref}(y_c|x)}-\beta\log\frac{\pi_\theta(y_r|x)}{\pi_{ref}(y_r|x)}\right)$

这条损失的直观解释其实很简单：相对参考模型，尽量把 chosen 的概率拉高，把 rejected 的概率压低。它本质上是一种“拉好答案、踩差答案”的对比学习。

换句话说，策略模型本身已经隐式携带了奖励信息，因此不再需要单独训练一个庞大的 Reward Model。

代价：离线分类很省，但探索能力会缩水

  • OOD 分布偏移：遇到训练集没覆盖的罕见问题，模型可能行为崩溃。
  • 奖励裕度爆炸：chosen 与 rejected 被越拉越开，模型越来越过度自信。
  • 多样性塌缩：回答风格变单一、模板化，复杂推理上限受限。
DPO 在主观风格对齐、开源微调和资源受限场景里非常强，但一旦任务需要实时探索新推理路径，静态数据集就会成为瓶颈。

GRPO：移除 Critic

PPO 的 Critic 在长 CoT 面前很吃力

对于需要几十步才能得出答案的数学题，模型刚写下第一行算式时，Critic 很难精准预判它最终会不会推导正确。

  • 长逻辑链下，早期 Token 的价值估计本身就很不稳定。
  • 如果基础模型是 671B 级别，再维护同体量 Critic，显存和带宽会直接逼近物理极限。
GRPO 的核心工程壮举：保留 RL 的主动探索，但彻底删除评论家网络。

替代方案：组内相对比较

不再依赖一个神经网络给绝对基线，而是对同一个 Prompt 并行采样一组完整推理路径，再在组内比较谁更好。

Prompt
给定同一个问题。
Group
并行采样 8 或 16 条回答。
Verifier
外部脚本给每条路径打分。
Mean / Std
计算组内均值和标准差。
Advantage
用相对分数替代 Critic 基线。

GRPO 的关键数学：Z-Score 相对评分

公式很简单，统计意义却极强

$$A_i=\frac{R_i-\operatorname{mean}(R_{1..G})}{\operatorname{std}(R_{1..G})}$$

优势不再来自 Critic 的绝对价值估计，而是来自这条轨迹相对于同组平均水平的“拉开程度”。

类比：大学期末考试“拉曲线”

如果一场《实变函数》考试极难，全班平均只有 30 分，那么 50 分虽然仍不及格，但在班里已经是极强正样本；相反，若题目很简单，全班平均 90 分，那么 80 分反而是负优势。

  • 题目难度高低不再直接干扰梯度尺度。
  • 同组总能分出相对更好的推理路径，避免梯度失去方向。
  • 罕见但优秀的 CoT 会被 Z-Score 放大并固化。
“内卷式机制”：模型不是和题目难度打架，而是在自己当前能力边界上不断突围。

GRPO + RLVR：用纯规则奖励点燃自我进化

DeepSeek-R1-Zero 的奖励系统极其朴素

准确性奖励
正确 = 1，错误 = 0
格式奖励
<think>...</think>

在数学与代码场景，很多时候几行验证脚本或形式化证明器就能替代黑盒神经奖励模型，这正是可验证奖励（RLVR）的威力。

结果：复杂推理行为从试错中涌现

  • Reflection：发现原路不通后主动换思路。
  • Self-correction：意识到错误后自发回滚重算。
  • Aha moment：长思维链中出现顿悟式跳跃。
这些能力并不是靠人工写标准答案“教”出来的，而是在规则奖励 + GRPO 相对评分下，通过强化学习自己涌现出来的。

报告给出的工业参考场景：8B 参数、20K 上下文。核心差异不是抽象优劣，而是显存账本能否落地。

PPO 的高成本来自 Actor、Critic、Reward、Reference 同时在场，还要维持优化器状态和大规模激活值。
GRPO 的革命性在于：去 Critic + 规则验证替代重奖励模型，再配合 LoRA 与激活检查点，显存可压缩约 90%。

前沿演进：DAPO / GSPO / ES

DAPO：修长度偏见

GRPO 会诱导模型通过“过短但不完整”的答案投机。DAPO 用不对称裁剪（Asymmetric Clip）和动态采样 / 截断屏蔽，逼模型靠真正的长逻辑链拿分，而不是靠偷步数。

GSPO：从 Token 提升到 Sequence

MoE 模型里，Token 级重要性采样方差很高。GSPO 把优化颗粒度抬到完整序列，更贴近“最终答案拿奖励”的本质，也更适配 MoE。

ES：无梯度黑盒路线

进化策略把模型看成黑箱：给权重注入高斯噪声，繁衍出一群子模型，只做前向评估，再聚合优胜变异方向更新。好处是几乎不吃反向传播显存，并且天然适合超大规模并行。

趋势不是追单一算法的绝对最优，而是围绕“任务类型 + 硬件边界 + 模型结构”寻找组合工程最优。

总结：对齐——数学与工程的双重博弈

从将文本生成定义为马尔可夫决策过程，
到用 Bradley-Terry 模型量化人类偏好；
从 PPO 的稳健牵引、DPO 的优雅降配，
再到 GRPO 激发的自我思考浪潮——

强化学习正在深刻重塑大语言模型的认知边界。
奖励黑客的幽灵提醒我们：对齐不仅是数学与算力的竞赛，更是目标设定与漏洞防御的工程博弈。
