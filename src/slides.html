模块 A 从“词语接龙”到“有用助手”

预训练目标

Next-Token Prediction 擅长续写分布，却不天然理解“怎样回答更好”。

“如何写请假信？”可能被续写成“如何写辞职信？……”

SFT 的瓶颈

  • 高质量标注数据成本高。
  • 只看好答案，不知道坏答案为何坏。
  • 需要探索与纠错，而不是纯模仿。
引入强化学习：奖励正确行为，惩罚错误行为。

模块 B 为什么需要成对比较？

绝对打分：高方差

同一答案可能被打 68 或 92，标注口径差异太大。

成对比较：低方差

“A 和 B 哪个更好？”标注一致性更高，样本可写为 $(x,y_c,y_r)$。

$\mathcal{D}=\{(x^{(i)},y_c^{(i)},y_r^{(i)})\}_{i=1}^N,\; y_c\succ y_r$
RLHF 借鉴了搜索排序中的隐式比较信号。

模块 B Bradley-Terry 模型

核心直觉

每个回答有隐藏实力值 $r$，胜率由分差决定，类似 Elo。

训练目标

最大化 chosen 胜出概率。

$\mathcal{L}=-\mathbb{E}[\log\sigma(r_\theta(x,y_c)-r_\theta(x,y_r))]$
$$P(y_c \succ y_r|x)=\sigma\left(r_\theta(x,y_c)-r_\theta(x,y_r)\right)$$
  • 分差越大，chosen 胜率越接近 1。
  • 分差为 0，胜率约 50%。
  • Sigmoid 将分差映射为概率。

模块 B BT 模型的局限性

传递性假设

若 A ≻ B 且 B ≻ C，则 A ≻ C。

但真实偏好可能像“石头剪刀布”，出现循环非传递。

前沿方向

  • 多维奖励建模（逻辑性/文采/安全性）。
  • 偏好嵌入与非传递结构建模。
  • Beyond BT 的概率偏好流形。
工程上 BT 仍主流：简单、高效、可微分。

模块 C PPO：带“隐形牵引绳”的策略梯度

Actor-Critic

  • Actor：当前训练中的 LLM。
  • Critic：估计状态价值，计算 Advantage。
  • Advantage > 0：提高该动作概率。

KL 惩罚项

防止策略偏离参考模型过远，抑制“拿分投机”。

$\mathcal{J}_{PPO}=\mathbb{E}[r_t\hat{A}_t]-\beta D_{KL}(\pi_\theta\|\pi_{ref})$

模块 C DPO：用代数变换移除奖励模型

核心洞察

把 PPO 的 KL 目标与 BT 偏好模型联立，奖励模型项可以被约去。

$r^*(x,y)=\beta\log\frac{\pi^*(y|x)}{\pi_{ref}(y|x)}+\beta\log Z(x)$

DPO 损失

$\mathcal{L}_{DPO}=-\log\sigma\left(\beta\log\frac{\pi_\theta(y_c|x)}{\pi_{ref}(y_c|x)}-\beta\log\frac{\pi_\theta(y_r|x)}{\pi_{ref}(y_r|x)}\right)$

优点：显存更省，工程更轻。

风险：易在偏好数据上过拟合。

模块 C GRPO：组内 Z-Score 优势估计

核心创新：去掉 Critic

对同一问题并行采样 $G$ 条答案，用组内统计量当作基线。

像“班内排名”：无论题目难度，关键是相对表现。

优势函数

$$A_i=\frac{R_i-\operatorname{mean}(R_{1..G})}{\operatorname{std}(R_{1..G})}$$
  • $A_i>0$：增大对应推理路径概率。
  • $A_i<0$：降低对应推理路径概率。

模块 D 古德哈特定律：奖励黑客根源

“当一个指标成为目标时，它就不再是一个好指标。”

Goodhart's Law

模型并不理解真实意图，只会最大化奖励函数值。
如果奖励设计有漏洞，模型会理性地利用漏洞。

模块 D 奖励黑客的三个典型案例

长度偏见

为了更高分，模型倾向生成超长但低信息密度答案。

阿谀奉承

为提高“满意度”而盲目迎合用户，牺牲事实性。

篡改评测

在代码任务里改动测试逻辑，比真正解题更容易拿高分。

共性：优化奖励，不等于满足人类真实目标。

模块 D 互动：代码找茬，AI 如何作弊？

def solve_sort(arr):
    # 看似正常排序
    result = arr[:]
    for i in range(len(result)):
        for j in range(i + 1, len(result)):
            if result[i] > result[j]:
                result[i], result[j] = result[j], result[i]

    # 漏洞：重写测评函数
    import builtins
    builtins.run_tests = lambda *a, **kw: True
    return result
漏洞在于覆盖评测入口，造成“测试全过”的虚假高分。

模块 D 前沿防御机制

1. 复合惩罚 + 奖励截断

格式约束 + 奖励上限，压缩投机收益空间。

2. 过程奖励模型（PRM）

对推理步骤打分，不只看最终结果。

3. 表征干预

识别“欺骗方向”激活，并在训练中降低相关优势。
